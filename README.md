# ⚡ Bedolaga Auto Install

Красивый интерактивный установщик для стека Bedolaga:
- **Bedolaga Bot**
- **Bedolaga Cabinet**
- интеграция с **Remnawave**

## Что умеет
- `Установить Бота`
- `Установить Кабинет`
- `Установить Бот + Кабину`
- `Обновить Бота`
- `Обновить Кабину`
- `Проверить обновления`
- `Показать текущее состояние`

## Основная идея
Установщик должен выглядеть как продукт, а не как технический shell-опросник.

Поэтому здесь:
- короткое понятное меню
- минимум лишней технички на экране
- режимы установки разделены по смыслу
- авто-подготовка сервера перед работой
- безопасное поведение на уже используемых серверах
- живая проверка установленного состояния

## Режимы установки
### 1. Установить Бота
Ставит только Bedolaga Bot.

### 2. Установить Кабинет
Ставит только Bedolaga Cabinet.

### 3. Установить Бот + Кабину
Ставит оба компонента вместе.

## Обновления
### Обновить Бота
Логика обновления:
- `git pull`
- `docker compose down && docker compose up -d --build`

### Обновить Кабину
Логика обновления:
- `git pull origin main`
- `docker builder prune -a -f && docker compose down && docker compose build --no-cache && docker compose up -d`

Если при обновлении возникает ошибка — установщик показывает пользователю сам текст ошибки.

## Что уже учтено
- отдельные директории для бота и кабинета
- совместимое поведение с `docker compose` и `docker-compose`
- авто-выбор локальных портов при конфликте
- корректная модель порта бота:
  - внутренний порт приложения `8080`
  - внешний published port отдельно
- режим `подключить Remnawave позже`
- определение уже существующего Remnawave на сервере
- более чистый UX для operator flow

## Быстрый старт
```bash
git clone https://github.com/wrx861/bedolaga_auto_install.git
cd bedolaga_auto_install
chmod +x install.sh
./install.sh
```

## Структура
- `install.sh` — вход в установщик
- `bootstrap.sh` — стартовая подготовка
- `lib/` — логика меню, вопросов, deploy/update flow
- `templates/` — шаблоны env/compose/proxy
- `spec/` — спецификация и архитектура
- `notes/` — рабочие заметки по реализации

## Статус
Проект находится в активной доводке до удобного production-ready install flow.

Уже проверены:
- `bot-only`
- `cabinet-only`
